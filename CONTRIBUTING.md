# Contributing to Linkaton

Thank you for your interest in contributing to Linkaton.

We welcome bug reports, feature requests, improvements, and code contributions.

## How to Contribute

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Commit your work
5. Open a Pull Request

## Branch Naming

Use clear branch names:


feature/add-authentication
fix/login-error
docs/update-readme


## Commit Messages

Linkaton follows Conventional Commits:


feat: add new feature
fix: fix a bug
docs: update documentation
refactor: improve code structure
chore: maintenance changes


Example:


feat: add URL shortening service


## Pull Requests

Before submitting a Pull Request:

- Make sure your code works
- Keep changes focused
- Update documentation if needed
- Explain what was changed

## Code Style

Please keep the code clean, readable, and consistent with the existing project style.

Thank you for helping improve Linkaton ❤️
